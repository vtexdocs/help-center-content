# Cross-Portal Alignment — Naming Decisions

The following section names were chosen with the Developer Portal's existing labels in mind.

**"Intelligent Search" — kept as canonical name**
The Dev Portal uses "Search" for the same product area — that is a Dev Portal misalignment, not a reason to rename the HC section. The correct canonical name is "Intelligent Search" (as used in the VTEX Admin UI), and the Dev Portal should align to it.

**"Shipping" — kept over "Logistics"**
The Dev Portal uses "Logistics" for its API-facing content (Fulfillment API, delivery promise). The divergence is intentional: "Shipping" is the merchant-facing term in the Admin UI; "Logistics" is the developer/API-facing term. Both are correct for their audience.

**"Pricing and Promotions" — "and" over ampersand**
The Dev Portal's troubleshooting filter already uses "Pricing and Promotions," so the HC proposal aligns to that form. Plain "and" also avoids special character encoding issues in metadata and sidebar rendering.
