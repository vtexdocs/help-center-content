## Implementation Order

When applying changes across the three language trees, use this sequence to minimize broken states:

1. Apply all changes to EN first (it is the source of truth for structure)
2. Validate EN navigation works in staging environment
3. Apply the same changes to ES, using the folder name equivalents from this document
4. Apply the same changes to PT, using the folder name equivalents from this document
5. Validate all 3 language trees in staging before merging to production

**Never apply changes to only one or two language trees.** All three must be updated in the same pull request or the CMS will show inconsistent navigation.
