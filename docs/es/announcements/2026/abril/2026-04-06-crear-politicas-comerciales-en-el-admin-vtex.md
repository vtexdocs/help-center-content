Ahora pueden crearse políticas comerciales en el **Admin VTEX**, lo que permite que usuarios con el rol **Owner (admin)** agreguen nuevas políticas de forma autónoma, sin necesidad de abrir un ticket.

> ℹ️ La creación de una nueva política comercial puede generar costos adicionales, según lo previsto en el contrato de cada cliente. Si tienes dudas, consulta tu contrato o ponte en contacto con nuestro [Soporte](https://supporticket.vtex.com/support). Todas las acciones de creación se registran en los logs de auditoría del Admin, lo que permite identificar al usuario que realizó la acción. Para más información, consulta [Cómo funciona el informe de seguridad](https://help.vtex.com/es/docs/tutorials/informe-de-seguridad).

¿Qué cambió?

La creación de políticas comerciales se realiza directamente en el **Admin**, haciendo clic en el botón `Nueva política comercial` en la interfaz para usuarios con rol **Owner (admin)**.

Los beneficios de esta actualización son:

- Mayor autonomía para crear y gestionar políticas comerciales.  
- Eliminación de la necesidad de abrir tickets para esta operación.

¿Qué se necesita hacer?

La funcionalidad ya está disponible para todas las cuentas VTEX. El único requisito es que el usuario tenga el perfil **Owner (admin)**. Para más información, consulta el artículo [Crear una política comercial](https://help.vtex.com/es/docs/tutorials/crear-una-politica-comercial).

> ⚠️ Recomendamos que cada usuario tenga solo los permisos necesarios para ejecutar sus actividades. Revisa los usuarios con el rol Owner y asegúrate de reservarlo únicamente para aquellos que realmente necesiten este nivel de acceso.
