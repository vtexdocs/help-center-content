## Definições

Uma __tabela de preços__ é um conjunto de preços de SKUs que pode ser aplicado a um determinado contexto. Essas tabelas guardam as informações dos preços que são exibidos em uma loja.

Podemos entender uma tabela de preços como uma entidade que armazena e entrega os __preços computados__ dos seus SKUs. 

## Relação entre Tabelas de preços e Políticas comerciais

Há uma diferença entre as tabelas de preços criadas para *[políticas comerciais](https://help.vtex.com/pt/tutorial/como-funciona-uma-politica-comercial--6Xef8PZiFm40kg2STrMkMV)* e tabelas de preços criadas para *outros contextos*. 

As tabelas de preços não associadas a políticas comerciais podem ser usadas em contextos como __promoções__ e __clusters de clientes__. As tabelas criadas para políticas comerciais *só podem ser associadas à própria política comercial*. Assim, a tabela de preço associada a uma política comercial informa o preço dos SKUs para esta política e não pode ser associada a outros contextos.

É importante ressaltar que os conceitos de __tabela de preços__ e __política comercial__ são *diferentes*. A política comercial é o *contexto* ao qual uma tabela de preços é associada.

## Exemplo

Imagine um cenário em que você queira criar uma estratégia de precificação para vender um SKU em três diferentes contextos. Para isso, é preciso criar três tabelas de preço - uma para cada contexto.

A ideia é ter um preço específico para: 

- aplicação de uma promoção; 
- um cluster de clientes;
- uma política comercial.

Você deve criar __duas tabelas de preço genéricas__, uma para o cluster de clientes e outra para a aplicação da promoção. Além disso, você deve inserir o preço desejado na __tabela de preço associada à política comercial__ em questão.
