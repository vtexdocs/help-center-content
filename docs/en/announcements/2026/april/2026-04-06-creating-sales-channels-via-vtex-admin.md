You can now create sales channels (formerly trade policies) through the **VTEX Admin**. This allows users with the **Owner (admin)** role to add new policies autonomously, without needing to open a ticket.

> ℹ️ Creating a new sales channel may generate additional costs as stipulated in your contract. If you have any questions, check your contract or contact our [Support](https://supporticket.vtex.com/support). All creation actions are logged in the Admin audit logs, allowing you to identify who performed the action. For more information, see [Understand the Security report](https://help.vtex.com/pt/docs/tutorials/entendendo-o-relatorio-de-seguranca).

What has changed?

Now you can create sales channels directly in the **Admin** using the `New sales` `channel` button in the user interface for users with the **Owner (admin)** role.

These are the benefits of this update:

- Greater autonomy to create and manage sales channels.  
- No longer dependent on submitting tickets for this operation.

What needs to be done?

The feature is available to all VTEX accounts. The only requirement is that the user must have the **Owner (admin)** role. For more information, see the article [Creating a sales channel (formerly trade policy)](https://help.vtex.com/pt/docs/tutorials/criar-uma-politica-comercial).

> ⚠️ We recommend giving users only the permissions required for their tasks. Review users with the Owner role and limit this access to those who truly need it.
