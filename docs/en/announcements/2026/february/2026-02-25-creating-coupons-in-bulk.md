The Promotions module will allow [creating coupons](https://help.vtex.com/docs/tutorials/creating-a-coupon-beta) in bulk. This update increases the supported volume and reduces manual operations when creating campaigns that require a large number of coupons.

## What has changed?

The bulk feature previously allowed creating a maximum of 1,000 coupons per operation, which limited campaigns that required a higher number of codes.

The limit has been increased to 50,000 coupons, allowing you to create larger batches automatically. To use the feature, enable the **Generate coupons in bulk** option in the **Promotions** module and enter the desired quantity.

Batches with more than 1,000 coupons aren’t created immediately. In these cases, processing continues in the background, so you can keep using the VTEX Admin for other activities. During this period, you can track progress in the coupon list, where a column indicates the `processing` status.

> ℹ️ Batches with more than 1,000 coupons can't be edited. If you want to change the information in a specific batch, you'll need to archive it and create a new batch with the updated information.

## What needs to be done?

This feature is in the testing phase (Closed Beta), available only to a selected group of accounts. To participate in this testing phase and validate the feature, contact the [VTEX Support team](https://support.vtex.com/).
