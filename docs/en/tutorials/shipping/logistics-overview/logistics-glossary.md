The glossary below contains definitions of terms related to the **Shipping** module.

| Term | Meaning | Learn more |
|------|----------|------------|
| **Additional shipping charge** | Extra costs that may be added to the [shipping rates](/en/docs/tutorials/shipping-rates). There are four types of additional charges: <br> - [Weight-based additional charge](/en/docs/tutorials/additional-shipping-costs#weight-based-additional-shipping-charge) <br> - [Price-based additional charge](/en/docs/tutorials/additional-shipping-costs#price-based-additional-charge) <br> - [Additional insurance charge](/en/docs/tutorials/additional-shipping-costs#additional-insurance-charge) <br> - [Additional shipping charge](/en/docs/tutorials/additional-shipping-costs#additional-shipping-charge) | [Additional shipping charge](/en/docs/tutorials/additional-shipping-costs) |
| **Cargo/Package splitting** | When the total weight calculation exceeds the limit accepted by the carrier, this feature allows you to split the items in the shopping cart and send them in more than one package. | - [Cargo split](/en/docs/tutorials/configuring-cargo-splitting) <br> - [Order split and delivery split](/en/docs/tutorials/order-split-and-delivery-split) |
| **Critical inventory** | Name for the low inventory alert. | [Critical inventory](/en/docs/tutorials/setting-up-the-critical-stock-alert) |
| **Cubic weight factor** | It is the ratio between the weight and the volume of a package. This factor helps estimate the package's mass and space required for delivery. | [Cubic weight factor](/en/docs/tutorials/understanding-the-cubic-weight-factor) |
| **Delivery** | In this option, the customer receives their orders at a given address. |  |
| **Delivery capacity** | The maximum number of deliveries in a period of time. | [Delivery capacity](/en/docs/tutorials/scheduled-delivery) |
| **Delivery channels** | Type of delivery: shipping and pickup points. The shipping option is when the customer receives the order at the given address, usually delivered by a carrier. The pickup point is where the product will be available for the customer to pick up. |  |
| **Flat rate (shipping)** | Base rate of the shipping rate. | [Shipping rates](/en/docs/tutorials/shipping-rates) |
| **Inventory** | List of SKUs in stock and their respective quantities. | [Inventory](/en/docs/tutorials/managing-stock-items) |
| **Loading dock** | Entity that links the warehouse to a shipping policy. | [Loading dock](/en/docs/tutorials/loading-dock) |
| **Loading dock rate** | **Additional cost** for handling products from the warehouse to the loading dock. This value is configured in [Warehouses](/en/docs/tutorials/managing-warehouses), in the **Loading dock relationship** section. | [Filling in warehouse fields](/en/docs/tutorials/managing-warehouses#filling-in-warehouse-fields) |
| **Marketplace** | Store responsible for selling the items, but not for shipping them. | [Marketplace](/en/docs/tutorials/what-is-a-marketplace) |
| **Modal** | Classification of the items' types to be delivered by the carriers. | [Modal](/en/docs/tutorials/how-does-the-modal-work) |
| **Package** | Refers to a set of items from an order that are wrapped together. A single order can have more than one package. | [Order split and delivery split](/en/docs/tutorials/order-split-and-delivery-split) |
| **Pickup points** | Physical locations for customers to pick up their orders. | [Pickup points](/en/docs/tutorials/pickup-points) |
| **Seller** | Stores that are responsible for shipping the orders, but not necessarily for selling them. | [Seller](/en/docs/tutorials/what-is-a-seller) |
| **White label sellers** | A seller that is not specified for the end customer. The customer does not know which store is responsible for shipping the product, only the marketplace. Generally used for franchise accounts. | [White Label Sellers selection](/en/docs/tutorials/white-label-sellers-selection) |
| **Shipment/package group** | Configuration for grouping packages for shipment to minimize preparation and pickup time. | [Shipment](/en/docs/tutorials/setting-up-shipment) |
| **Shipping** | A package can be **delivered** (to the customer's address) or **picked up** (by the customer at a certain location). |  |
| **Shipping policy** | It describes prices and deadlines agreed upon by the store and the carrier. | [Shipping policy](/en/docs/tutorials/shipping-policy) |
| **Shipping rate template** | A spreadsheet where you enter shipping cost options offered by a specific carrier. | [Shipping rate template](/en/docs/tutorials/shipping-rate-template) |
| **Shipping rates** | The costs set up in [Shipping policies](/en/docs/tutorials/shipping-policy). Composed of the flat rate and additional shipping costs. | [Shipping rates](/en/docs/tutorials/shipping-rates) |
| **Shipping simulator** | A tool for shipping simulation. It analyzes SKU availability, carriers, and pickup points. | [Shipping Simulator](/en/docs/tutorials/shipping-simulation) |
| **SKU** | Stock Keeping Unit. The basic unit of a product in the catalog (e.g., product "shirt" with SKU "blue shirt M size"). | [SKU](/en/docs/tutorials/what-is-an-sku) |
| **SLA** | Service Level Agreement. In VTEX logistics, it means the agreement between the store and the customer on the estimated delivery time. This estimate considers shipping and loading dock policies. | [SLA](https://developers.vtex.com/vtex-rest-api/reference/calculatesla) |
| **Total shipping cost** | The final shipping cost, considering fixed, additional, and dock costs. | [Total shipping cost](/en/docs/tutorials/total-shipping-cost) |
| **Trade policy** | A set of business settings for adjusting strategies (currency, country, shipping, pricing, customers, catalogs, and payment methods). | [Trade policy](/en/docs/tutorials/how-trade-policies-work) |
| **Warehouse** | The storage location of the store's goods. | [Warehouse](/en/docs/tutorials/warehouse) |
