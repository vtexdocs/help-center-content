## Sumário

<div class="alert alert-info">
  <p>Este problema conhecido foi traduzido automaticamente do inglês.</p>
</div>


Quando o marketplace tem muitos vendedores e usa as planilhas de mapeamento para a maioria deles, a UI de mapeamento de catálogo pode ser muito pesada e, cada vez que o comerciante tenta acessá-la, leva muito tempo para carregar.
Além disso, a usabilidade é prejudicada.

## Simulação



1. A conta precisa ter uma grande quantidade de vendedores;
2. Para cada vendedor, adicione-o à área de mapeamento da interface de mapeamento do catálogo;
3. Verifique se o tempo de carregamento dessa página está aumentando e se a usabilidade não está boa.



## Workaround


Não há solução alternativa para o mapeamento de categorias e marcas. Também é possível mapear a especificação usando a API:
https://developers.vtex.com/docs/api-reference/marketplace-apis-suggestions#put-/suggestions/configuration
