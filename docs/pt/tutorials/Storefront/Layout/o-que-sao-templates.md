Templates são os arquivos que contêm o código das páginas do seu web site. Eles são responsáveis, dentre outras coisas, por determinar como as informações vão aparecer na tela (inclusive a posição dos [placeholders](/pt/tutorial/o-que-sao-placeholders)).

Na plataforma VTEX, os templates são divididos em dois tipos:

- __HTML Templates (ou templates de página)__: são eles que estruturam o código HTML e referenciam arquivos JavaScript e CSS para serem usados na páginas do web site.
- __Shelves Templates__: definem a estrutura e o conteúdo das prateleiras, elementos que agrupam produtos a partir de critérios de exibição e podem ser usados em mais de uma página.

Essa divisão existe para facilitar a organização dos templates e, consequentemente, do código do seu web site. Imagine que você quer alterar o código de uma prateleira que aparece em várias páginas diferentes. É muito mais prático editar um único arquivo do que acessar os templates que contêm os códigos de cada uma delas.

Além desses dois tipos básicos, há ainda uma subdivisão dos __templates de página__, que são chamados de __Sub Templates__. Eles são, a exemplo das prateleiras, elementos que vão aparecer na maioria (ou em todas) as páginas do seu web site, como footers, menus, headers e outros. Por isso, também possuem um repositório próprio.

E falando em organização, você encontra todos os templates no módulo **Storefront > Layout > CMS**, reunidos em pastas de acordo com seu tipo (HTML ou Shelves). Os __Sub Templates__ estão na pasta de mesmo nome, que fica dentro do diretório __HTML Templates__. Nesses locais, você vai poder criar novos templates e editar os que já existem.
