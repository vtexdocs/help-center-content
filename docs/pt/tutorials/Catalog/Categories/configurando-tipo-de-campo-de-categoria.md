Este artigo faz parte das configurações de [Categoria](/pt/tutorial/cadastrando-categoria) &gt; [Campo de Produto](/pt/tutorial/criando-um-campo-de-produto) &gt; [Campo de SKU](/pt/tutorial/criando-campo-de-sku).

Os Tipos de Especificação podem variar entre Texto, Texto Grande, Número, Combo, Radio, CheckBox, Texto Indexado e Texto Grande Indexado.

Cada um atenderá a uma demanda diferente, conforme suas particularidades, descritas abaixo:

**Texto**

Preenchimento do conteúdo como texto livre, mas não indicado para grandes estruturas de HTML, como por exemplo para informar sobre os componentes que compões a entrega do Produto.

**Texto Grande**

Preenchimento do conteúdo como texto livre, indicado para grandes estruturas de HTML, como por exemplo para Tabelas de Medidas específicas &#8211; hoje muito usado no segmento de moda para um número de manequim maior.

**Número**

Preenchimento do conteúdo apenas com números inteiros, como por exemplo para se especificar numerações de sapatos (39, 41, 42...).

**Combo**

São criados normalmente dois valores pré-definidos neste campo. Para, no cadastro do SKU, se ter um box com uma combo de opções para se definir os componentes, como por exemplo, para a voltagem de algum produto eletrétrico, que irá variar apenas entre 110v e 220v.

**Radio**

São criados diversos valores pré-definidos neste campo. Para, no cadastro do SKU, se ter um uma grade de opções que irá variar de SKU para SKU. Como por exemplo para a definição de Cor dos componentes (vermelho, amarelo, azul, preto,&#8230;).

**CheckBox**

Aqui se tem apenas valores de ‘true’ e ‘false’. Isto é, será um campo onde se irá ativar ou inativar seu valor. Como por exemplo a flag de “Outlet”, que será marcada apenas quando com o produto em queima de estoque para a saída definitiva de linha.

**Texto Indexado e Texto Grande Indexado**

Seguem tecnicamente a mesma regra dos campos de Texto citados acima, porém por estes serem classificados como “Indexado”, o valor de seu preenchimento será voltado também para o indexador do catálogo, de modo que passam a influenciar nos resultados de busca dos clientes na loja. Veja mais detalhes em [Como funciona a busca da VTEX?](http://help.vtex.com/tutorial/como-funciona-a-busca-da-vtex/ "Como funciona a busca da VTEX?").
