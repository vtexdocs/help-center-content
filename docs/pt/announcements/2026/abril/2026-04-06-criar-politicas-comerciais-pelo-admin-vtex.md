Disponibilizamos a criação de políticas comerciais pelo **Admin VTEX**, permitindo que usuários com o perfil **Owner (admin)** adicionem novas políticas de forma autônoma, sem a necessidade de abrir um ticket.

> ℹ️ A criação de uma nova política comercial pode gerar custos adicionais, conforme previsto no contrato de cada cliente. Em caso de dúvidas, consulte seu contrato ou entre em contato com o [suporte](https://supporticket.vtex.com/support). Todas as criações são registradas nos logs de auditoria do Admin, permitindo identificar qual usuário realizou a ação. Para mais informações, consulte [Como funciona o relatório de segurança](https://help.vtex.com/pt/docs/tutorials/entendendo-o-relatorio-de-seguranca).

## O que mudou?

A criação de políticas comerciais passa a ser realizada diretamente pelo **Admin**, por meio do botão `Nova política comercial` na interface para usuários com o perfil **Owner (admin)**.

Os benefícios dessa atualização são:

- Maior autonomia para criar e gerenciar políticas comerciais.  
- Eliminação da dependência de abertura de tickets para essa operação.

## O que precisa ser feito?

A funcionalidade já está disponível para todas as contas VTEX. Basta que o usuário tenha o perfil **Owner (admin)**. Para mais informações, acesse o artigo [Criar uma política comercial](https://help.vtex.com/pt/docs/tutorials/criar-uma-politica-comercial).

> ⚠️ Recomendamos que cada usuário tenha apenas as permissões necessárias para executar suas atividades. Revise os usuários com o perfil Owner e mantenha esse perfil restrito aos usuários responsáveis por essa função.
